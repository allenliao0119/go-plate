I need to create a system specification document for a restaurant self-pickup ordering platform (similar to online food ordering, but customers pick up their own orders).

【System Overview】
- Platform Type: Restaurant online ordering with self-pickup
- Target Users: Customers and Restaurant Owners
- Core Value: Convenient online ordering, skip the queue, scheduled pickup

【Main User Roles and Functions】

1. Customer:
   - Browse restaurants by location/category
   - View restaurant details and menus
   - Add items to cart with customization options
   - Select pickup time slot
   - Make online payment
   - Receive order confirmation and pickup notifications
   - Track order preparation status
   - Pick up order at restaurant
   - Rate and review restaurants
   - View order history

2. Restaurant Owner:
   - Manage restaurant profile (name, address, hours, photos)
   - Create and manage menu items (name, price, description, images)
   - Set available pickup time slots
   - Receive real-time order notifications
   - Accept/reject orders
   - Update order status (confirmed, preparing, ready for pickup)
   - Mark order as completed when customer picks up
   - View sales reports and analytics
   - Manage business hours and holidays

3. Platform Administrator (optional, can be simplified):
   - Approve restaurant applications
   - Manage user accounts
   - Handle disputes and refunds
   - View platform analytics

【Key Features to Include】

Core Functions:
- User authentication (email/phone registration, social login)
- Restaurant search and filtering
- Shopping cart management
- Order placement and confirmation
- Payment integration (credit card, digital wallets)
- Order status tracking
- Notification system (email, SMS, push notifications)
- Rating and review system
- Order history
